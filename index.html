发表于 2021-01-30 更新于 2021-01-31 分类于 炼丹 Disqus：

论文地址

引入

作者希望用神经网络来解决SAT问题，提出了NeuroSAT模型。

NeuroSAT在训练过程中对每个问题的监督只有“可满足”和“不可满足”两种状态。测试时，每次给一个SAT问题P，NeuroSAT就会给出这个问题是否可满足，而且如果一个问题被判定为可满足，我们几乎总是可以从激活函数中找到一组合法的值代入。

他们又对模型进行了修改，输入的是有少量矛盾的不可满足问题，让模型自己去找到矛盾并改正，他们把这个修改叫NeuroUNSAT。

问题建立

SAT问题的定义：存在一组值代入后使得公式P为真，那么P是可满足的；否则P是不可满足的。

模型

SAT问题都具有比较简单的句法，因此用RNN之类的东西可以直接encode进向量空间里。但是，命题逻辑的语义里包含了丰富的不变性，如排列不变性（$P\wedge Q = Q \wedge P$）和否定不变性（$P$和$\neg P$可满足性相同），这些不变量用句法级别的encoding会被忽略。

由此作者提出了他们的架构NeuroSAT，既保留了排列不变性，也保留了否定不变性。对于一个SAT问题，每个字面量和每个子句(clause)都被encode为无向图中的一个节点，如果某个字面量出现在了一个子句中，那么把他们对应的节点连起来。NeuroSAT通过不断地把“信息”在图中传来传去对向量空间表示进行不断的精化。在任何一个时间点，每个字面量和每个子句都有一个embedding。每次迭代由两部分组成：首先，每个子句都接收到它的邻居（字面量）节点的信息，并且依据这个信息更新自己的embedding。然后，每个字面量都接收到它的邻居（子句）和它的补节点（对于字面量$P$，它会收到来自$\overline{P}$的信息，反之亦然）的信息，并且依据这个信息更新自己的embedding。

左半部分是迭代的第一步，右半部分是第二步。

具体地来讲，模型用两个向量$L_{init}, C_{init}$（都是$\mathbb{R}^d$）初始化，包含三个多层感知机$L_{msg}, C_{msg}, L_{vote}$和两个有Layer-Norm的LSTM $L_u, C_u$。

在每个时间点$t$，我们都用一个矩阵$L^{(t)}\in \mathbb{R}^{2n\times d}$来表示字面量的embedding（一共n个字面量，加上补就有2n个，每个的embedding维度都是d），一个矩阵$C^{(t)}\in \mathbb{R}^{m\times d}$来表示子句的embedding（一共m个子句，每个的embedding维度都是d）。他们分别用$L_{init}$和$C_{init}$堆起来进行初始化。同时，还有LSTM使用的隐状态$L^{(t)}_h$和$C_h^{(t)}$（尺寸和上面的对应），初始化为零矩阵。

定义矩阵$M$为图的关联矩阵（原文叫邻接矩阵，但我记得离散2里把点和边的连通性矩阵叫关联矩阵，anyway），即$M_{ij}=1$当且仅当第$i$个字面量$l_i$出现在子句$c_j$中。

定义Flip操作，每次接受一个矩阵L，将L中每个字面量对应的行替换为它的否定对应的行（即如果矩阵L的四行分别对应$P, Q, \neg P, \neg Q$，那么Flip后的矩阵四行分别对应$\neg P, \neg Q, P, Q$）然后输出。

一次迭代过程表示如下（注意和上面文字描述的版本相对应）：

第一行是更新子句矩阵，用关联矩阵去乘$L_{msg}$作用下的$L^{(t)}$，每个子句就只受到它相关的字面量的影响。

第二行是更新字面量矩阵，除了隐状态$L_h^{(t)}$和$M\text{C}_{msg}(C^{t+1})$以外，他还引入了Flip(L(t))来引入否定不变性。

在$T$次迭代后（一开始设置的最长迭代步数），用最后的一个全连接$L_{vote}(L^{(T)})$来得到结果向量$L_*^{(T)}\in\mathbb{R^{2n}}$，每个字面量得到一个标量值；再将这个向量取平均得到$y^{(t)}=\text{mean}(L_*^{(T)})\in \mathbb{R}$，通过训练使得$y^{(T)}$和真实标签$\phi(P)$之间的sigmoid cross-entropy loss最小。

这个模型的好处：

  • 在架构设计上引入了排列不变形和否定不变性
  • 与问题规模无关，可以处理各种规模的问题

训练数据

作者希望让模型有比较好的泛化能力，而不只是从某些领域内学得很好而其他地方很烂。

因此作者创建了一个问题集$\mathbf{SR}(n)$，由包含$n$个变量的SAT问题对组成，每对中一个是可满足的，一个是不可满足的，而且他们只在一个子句中对一个字面量取否定就可以相互转换（换言之，加上/去掉一个$\neg$，这两个问题就一模一样）。

预测可满足性

首先在$\mathbf{SR}(40)$上做（虽然这种东西对于现代SAT求解器来说很容易，但在ML方面来看仍然是困难的）。

朴素LSTM直接歇逼，准确率不足50%（即不如瞎蒙）；MiniSAT求解器平均需要backjump 10次（这意味着拆十次存在量词/全称量词），做超过100次基本逻辑推理。

NeuroSAT用d=128，每个多层感知机都有3个隐层（也即一共5层？），Layer-Norm的参数选$10^{-10}$，跑26次迭代，用ADAM优化器，学习率$2\times 10^{-5}$，clip rate取0.65来全局地剪切梯度（这是不是那个dropout？）。为了方便训练，他们训的是$\mathbf{SR}(\mathbf{U}(10, 40))$，但在$\mathbf{SR}(40)$上测试，得到了85%的准确度。接下来作者试图了解如何从被认为是可满足的问题中decode出一组解。

解码满足条件

上图中，白色代表中立，蓝色表示倾向于不可满足，红色表示倾向于可满足，颜色深浅表示置信度高低。

因此我们可以看出，一开始网络整体以较低的置信度倾向于不可满足，但从一个转折点开始，有的字面量开始以较高的置信度倾向于可满足，而且很快所有字面量都开始以高置信倾向于可满足，随后网络收敛。

在正确地被判定为可满足的问题里，NeuroSAT的表现都差不太多。在NeuroSAT判定为不可满足的问题里，自始至终都是低置信的不可满足——它从来不会判定一个问题是高置信地不可满足。这意味着NeuroSAT会搜寻一个可满足性的【凭证】，只要找到就会开始判定可满足。

作者认为虽然引入了n个字面量的否定，但是从结果来看NeuroSAT对每个字面量还是只用了一个bit来encode。

如果把高维的字面量Embedding用PCA（主成分分析）降维投影到二维，那么可以发现，上文对应的转折点前红点（支持可满足）和蓝点（支持不可满足）是无法用一条线进行划分的，而转折点后则是可以轻易划分的。

因此作者认为，在转折点后挑一个时间点做PCA，然后红蓝点就分别对应了字面量取值的1和0，由此就找到了一组可行解。

在$\mathbf{SR}(\mathbf{U}(10,40))$上训练的网络，在$\mathbf{SR}(200)$上也可以做，只是最好最好就是25%了……

不同类型的问题

用不同的生成随机图的方法整了一堆10个点、17条边的图，出了好多不同类型的问题比如图染色、domaining-set problem（我没查到这是啥??）、完全图搜索、点覆盖等等，最后保留了4888个问题。

跑了512次迭代的NeuroSAT达到了85%的准确度。

寻找UNSAT core

在$\mathbf{SR}(\mathbf{U}(10,40))$上训练的NeuroSAT找可行解比较好，但不会给出不可行的证明，它没有暴力遍历可能性，而是随机地去撞大运。

但是作者又构造了一种新的数据集叫$\mathbf{SRC}(n,u)$，和$\mathbf{SR}(n)$类似，但是每个不可实现的问题都有一个已经不可满足的子句$u$（叫unsat core）（这里unsat core是只要把一个$\neg$加上或去掉就会变回可满足的子句）

在这种集合上训练的NeuroSAT（叫做NeuroUNSAT）会一上来就高置信地支持不可满足，然后在一个转折点开始低置信地支持可满足（如果判定可满足）或者在某个/某些字面量上以极其高的置信支持不可满足（如果判定不可满足）。

Introduction

目前的知识库自动补全SOTA模型使用神经链预测模型(neural link prediction model)来学习符号的分布向量表示。向量表示允许模型通过相似性来对未知的事实进行泛化。

然而，神经链预测模型对于需要多个步骤进行预测的情况预测不佳（A是B的父亲，B是C的家长=>A是C的祖父或祖母）。与之相反，符号定理证明器(symbolic theorem prover)正好可以处理这类问题。进一步，归纳逻辑程序设计(Inductive Logic Programming)利用这些证明器来从数据中学到可解释的规则并且在完成知识库的过程中进行应用。 但是，symbolic prover在subsymbolic表示的学习和相似度比较上能力比较差，这限制了他们对相似但不完全相同的询问进行泛化的能力。

尽管逻辑和机器学习的联系的问题已经由统计相关学习方法(statistical relational learning approaches)所解决了，以往的模型要么不支持次符号表示的推理，要么不是端到端训练出来的。神经多跳推理模型(neural multi-hop reasoning chain)通过将推理链encode到向量空间上去（也就是：在与答案进行比对之前，先迭代地将subsymbolic representation进行refine），在某种程度上解决了这个问题。但是它也有问题：可解释性差，而且难以显式地将领域内知识（例如作为某条规则）传递进去。

受到Prolog的反向链式算法和使用次符号表示的可微的归一化操作的方法，作者提出了Neural Theorem Prover。

Background

概念简介：

一个atom包含一个谓词和若干个项，因为我们考虑的是无函数的一阶逻辑，每个项只能是常量或者变量。例如，$[grandfatherOf, \textbf{Q}, BART]$是一个包含了谓词grandfatherOf和两个项（变量Q和常量BART）的atom

一个rule形如$H\text{:-}B$，其中H是一个atom，B是一系列atom的合取（可能为空），（代表B推出H）

一个不含变量的rule叫做ground rule，一个B为空的rule叫做fact

一个substitution set形如$\psi={X_1/t_1, X_2/t_2,…,X_N/t_N}$，其中$X_i$是变量，而$t_i$是用于替换变量的值。将一个substitution set应用于一个atom即为将这个atom中的所有变量替换为substitution set中对应的替换值。

给定一个query（也称作goal），以$[grandfatherOf, \textbf{Q}, BART]$为例，我们可以用Prolog’s backward chaining algorithm来找到Q适合的替换值。

总地来说，这个算法基于两个函数，OR和AND。

首先，OR遍历所有的rule（包括fact），将每个rule的H和goal进行合并（谓词相同才可以合并，用goal里的对象对应地替换rule里的对象），同时更新substitution set。因为只要有一条rule能够成功合并就足够了，所以这个操作称作OR。

当合并成功后，OR会调用AND来证明rule的body中的所有rule（也叫作subgoal）。AND首先将substitution set应用于第一个atom，然后继续调用OR来证明他；随后对每一个body中的atom都做相同的事情。只有所有的atom都被证明才可以说明goal被证明，所以这个操作叫做AND。

举一个例子：

goal是$[grandfatherOf, \textbf{Q}, BART]$。

有一条rule是$[grandfatherOf, \textbf{X}, \textbf{Y}]\text{:-}[[fatherOf, \textbf{X}, \textbf{Z}],[parentOf, \textbf{Z}, \textbf{Y}]]$

这样OR操作合并后得到的subgoal就是$[fatherOf,\textbf{Q},\textbf{Z}]$和$[parentOf,\textbf{Z}, BART]$，然后再用AND操作分别去证明。

个人感觉英文wiki上的算法说明更明确易懂……

Differentiable Prover

我们将NTP的构建表述为一系列module的构建。每个module输入一些离散的对象（如atom和rule）和一个proof state，同时输出若干新的proof state。

一个proof state$S = (\psi, \rho)$由substitution set $\psi$和神经网络$\rho$构成。其中$\psi$是截至目前的substitution set，$\rho$会输出一个（部分）证明的success score（是实数）。尽管离散的对象和substitution set都只在建立网络的过程中用到，一旦网络建立完成，在训练和测试的时候对于不同的proof我们就可以计算连续的proof success score。

module由离散的对象和substitution set实例化。他们构成了一个表示（部分）证明的success score的神经网络，同时递归地实例化子module来继续证明。

所有module都共享以下声明：$D\times S\to S^N$，其中D是控制网络构建的域，S是proof state的域，N是输出proof state的数目。进一步地，我们记$S_\psi$为proof state S对应的substitution set，$S_p$为它对应的神经网络

合并模块

在backward chaining中，两个atom的合并是核心（例如，一个是我们要证明的goal，另一个是一条rule的rule head）。在朴素的backward chaining中，两个非变量的符号要检查是否一致——一致才可以合并，否则直接退出。然而，我们希望的是让它对于不完全一致而含义类似（e.g. grandfatherOf和grandpaOf）的也进行合并，因此我们通过神经网络来衡量两个符号在向量空间中的相似性。

unify模块就用来更新substitution set并创建一个可以比较非变量符号的向量表示的相似性的神经网络。它的签名形如$L\times L\times S\to S$，其中L是由所有项的列表构成的集合（L中的元素都是包含若干项的列表），S是proof state的定义域。unify模块接受两个atom（表示为项的集合形式）和一个上游proof state，返回一个下游proof state。

unify模块首先遍历两个atom并且比较这些符号，如果一个符号是变量那么更新substitution set；否则将他们通过一个径向基函数(radial basis function)核，作者选取的超参数$\mu=\frac{1}{\sqrt{2}}$

其中$S’$是新proof state，$\nu$是变量符号的集合，$h/g$表示用值g替换变量h（也就是更新substitution set的过程）。这里的$S_\rho’$实际上是一个神经网络，min里面的值是【如果将这组输入喂给神经网络那么神经网络会输出的结果】。

注意到$S_\rho$的值一定落在$(0,1]$上。

OR模块

or模块的签名为$L\times \mathbb{N}\times S\to S^N$，其中L是所有goal atom的集合，$\mathbb{N}$是正整数集（指明神经网络最长的推理长度），S是输入proof state，输出若干个proof state。

and模块

首先定义一个辅助函数substitute，只要可能就会在atom中将变量替换（能换的尽量换）

and模块的签名签名为$L\times \mathbb{N}\times S\to S^N$，其中L是所有goal atom的集合，$\mathbb{N}$是正整数集（指明神经网络最长的推理长度），S是输入proof state，输出若干个proof state。（和or一样）

如果S是FAIL，或者剩余的步数是0，那么and操作都会失败。如果atom集合为空，那么直接将输入proof state输出。否则将atom集合中的每个元素拿出来，经过substitute替换后用or去递归建立神经网络。hexo

概率统计和数据结构这种正派硬课我还是认真听了、也有充分能力学好的，至于张雷和李列明的两门司马课程我还是希望它们尽快消失在历史的长河中。
